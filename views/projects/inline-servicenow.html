AI at ServiceNow

Skill Builder Platform

Problem Definition

ServiceNow needed a comprehensive platform to help users build and deploy AI skills across the platform. The existing approach was fragmented, with users struggling to discover, configure, and manage AI capabilities effectively.

Hypothesis

A centralized skill builder platform would enable users to discover, configure, and deploy AI capabilities more intuitively, reducing time-to-value and increasing adoption rates.

Method

User research through interviews and usability testing. Prototyping with Figma and After Effects for interaction design. Technical validation through LangChain and Python implementations.

Result and next steps

Created an intuitive skill builder interface with guided setup flows, orchestration capabilities, and runtime management. Users can now discover and deploy AI skills in minutes rather than hours.

Guided Setup Flow

Streamlined Configuration

Designed a guided setup flow that walks users through AI skill configuration step-by-step. The interface reduces cognitive load by breaking complex AI setup into manageable, contextual steps.

Implemented progressive disclosure patterns and contextual help to guide users through technical configurations without overwhelming them.

Hypothesis

Users will be able to configure AI skills more successfully with guided, contextual assistance rather than navigating complex technical documentation.

Method

Usability testing with both technical and non-technical users. A/B testing of different flow patterns to optimize completion rates and user satisfaction.

Result and next steps

Configuration success rates increased by 40% with the guided setup flow. Users reported higher confidence in deploying AI capabilities and reduced support tickets.

AI Orchestration

Designed an orchestration interface that allows users to chain multiple AI skills together, creating complex workflows and automation sequences.

Hypothesis

Users will be able to create sophisticated AI workflows by connecting individual skills in a visual, intuitive manner, reducing the need for complex scripting or technical expertise.

Methods

User interviews with workflow automation experts. Prototype testing with drag-and-drop interface patterns. Technical validation through LangChain integration.

Results and next steps

Users successfully created complex AI workflows 60% faster than traditional methods. The visual orchestration interface reduced technical barriers and increased adoption among non-technical users.

Prompt Design & Evaluation

Advanced Prompt Engineering

Developed sophisticated prompt design tools that help users create, test, and optimize AI prompts for better performance and accuracy.

Hypothesis

Users will be able to create more effective AI prompts through guided design tools and real-time evaluation feedback, improving AI output quality and reducing iteration cycles.

Methods

A/B testing of different prompt design interfaces. User studies with prompt engineering experts and novices. Performance metrics analysis across different prompt variations.

Results and next steps

Prompt effectiveness improved by 35% with the guided design tools. Users reported faster iteration cycles and higher confidence in their AI implementations.

Runtime Management

Designed comprehensive runtime management tools for monitoring, debugging, and optimizing AI skill performance in production environments.

Hypothesis

Users will be able to monitor and optimize AI performance more effectively with real-time analytics and debugging tools, leading to better system reliability and performance.

Methods

Performance monitoring and analytics implementation. User testing with DevOps and AI operations teams. Iterative design based on real-world usage patterns.

Results

AI skill performance monitoring reduced debugging time by 50% and improved system reliability. Users gained better insights into AI behavior and optimization opportunities.
